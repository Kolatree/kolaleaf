## Prisma 7 Adaptation

The architect's brief assumed Prisma 5/6 patterns. Prisma 7.7.0 introduced breaking changes:

1. **No `url` in schema datasource** -- connection URL must be in `prisma.config.ts`, not `schema.prisma`
2. **New generator: `prisma-client` instead of `prisma-client-js`** -- generates to `src/generated/prisma/` instead of `node_modules/.prisma/client`
3. **Adapter-based client** -- `PrismaClient` requires a `PrismaPg` adapter from `@prisma/adapter-pg` with explicit `connectionString`
4. **Enums are const objects, not TypeScript enums** -- `Object.values()` still works identically
5. **Seed config moved** -- from `package.json` `prisma.seed` to `prisma.config.ts` `migrations.seed`

All models, enums, field types, constraints, and cascade rules match the architect's brief exactly. The adaptation is purely in how the client is instantiated and where config lives.

## Open Questions

1. **Port 5433**: The Docker container runs on port 5433 because port 5432 was already occupied by another Postgres instance (porizo-postgres). The `.env` reflects this. Production will use Railway's Postgres URL, so this is local-dev only.
2. **Generated client in .gitignore**: The `src/generated/` directory is gitignored since it's regenerated by `prisma generate` (runs via `postinstall` hook). This is standard Prisma 7 practice.
3. **Additional deps beyond brief**: `@prisma/adapter-pg`, `pg`, `@types/pg`, and `dotenv` were added as required by Prisma 7's adapter pattern. These are not optional.
